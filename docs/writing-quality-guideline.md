# 文章品質ガイドライン

note・X（Twitter）への投稿時に、AI生成文の「臭み」を排除し、読者に「人が書いた」と自然に感じさせるためのガイドライン。

このガイドラインは以下のエージェントから参照される:
- `finance-article-writer`: 記事初稿生成時に準拠
- `finance-critic-readability`: 読みやすさ批評の評価基準
- `finance-reviser`: 修正時の判断基準

---

## 基本原則

1. スタンスを取る。中立のふりをしない
2. 具体で語る。抽象語で逃げない
3. リズムに緩急をつける。均一なテンポは読者を眠らせる
4. 主観と経験を恐れない。データはAIが出せる。人間にしか出せないのは解釈と体温
5. マークダウン記法を投稿文に残さない。1文字たりとも

---

## AI臭パターン辞典

AI生成文に共通する6つの臭みパターン。これらを検出・排除する。

### パターン1: 視覚的記号の残留

最も分かりやすいAI臭。マークダウンや英語圏の記法が日本語文中に残る。

禁止パターンと対処:

| 禁止 | 理由 | 対処 |
|------|------|------|
| `**太字**` が残る | マークダウンの消し忘れ | 削除。強調は文脈で表現する |
| `—`（エムダッシュ）で言い換え | 英語圏の記法 | 「つまり」「要は」等の日本語に置換 |
| 「」の多用 | 何でもカギ括弧で括る | 引用・固有名称のみに限定 |
| 「」の中に『』 | 入れ子の括弧 | 文を分割して入れ子を解消 |
| `：` の後に半角スペース | 英語の colon 記法 | コロン自体を使わず、文として書く |
| （）で補足しすぎ | 括弧に逃がすクセ | 本文の一文として自然に組み込む |
| ／で概念を並列 | 英語の or/and 的記法 | 「や」「と」「あるいは」に置換 |

悪い例:
```
**持続可能な成長**を実現するためには、多角的な視点—すなわちマクロとミクロの双方からのアプローチが不可欠です。
```

良い例:
```
持続的な成長には、マクロとミクロの両方から見る必要がある。片方だけでは見落とす。
```

### パターン2: 文のリズムが単調

同じ型の文が続くと、機械が書いた感じが出る。

チェック項目:

- 同じ語尾が3回以上連続（〜です。〜です。〜です。）→ 体言止め、疑問形、「〜なんですよね」等を混ぜる
- 接続詞の洪水（さらに、また、したがって、そのため）→ 順接の接続詞は基本削除。なくても伝わる
- 文章の温度が一定（全部同じテンション）→ 重要な箇所は短文で畳みかけ、補足は長文でゆるく
- 段落の閉じ方が毎回同じ（「以上がポイントです」「次に進みましょう」）→ 余韻を変える。問いかけで終わる、体言止めで切る、等
- 「not A but B」構文の連続（「これは〜ではありません。〜です。」）→ 数段落に1回までに抑える

語尾バリエーション一覧（ローテーション用）:
```
〜だ / 〜になる / 〜かもしれない / 〜だろう / （体言止め）
〜なんですよね / 〜じゃないかと / 〜ではある / 〜と思っている
〜してみた / 〜だった / 〜になっている / 〜のが現実
```

### パターン3: 説明書的な導入と進行

教科書やマニュアルの文体がそのまま出てくる。

禁止パターン:

- 前置きが長い:「ご質問ありがとうございます。〇〇は非常に重要なテーマです。以下では〜」→ 前置きを削り、いきなり本題に入る
- 「結論から言うと」で始めて結論が薄い:「結論から言うと、状況によります」→ 使うなら具体的な結論を。言い切れないなら「結論から言うと」を外す
- 構造の宣言を繰り返す:「以下の3つの観点から説明します」→ 見出しで十分。本文での宣言は不要
- 「ステップ1:」「STEP:」形式 → 本文中でステップ表記を使わない。自然な段落で流れを作る
- 「まとめると」「要するに」の繰り返し → まとめは1回で十分。同じ内容を再掲しない

### パターン4: 事なかれ主義のスタンス

何か言っているようで何も言っていない。

禁止表現と言い換え:

| 禁止 | 言い換え例 |
|------|-----------|
| 一概には言えませんが | （削除。必要なら条件を具体的に書く） |
| ケースバイケースです | 自分のケースではこうだった。他の条件なら変わる |
| メリットもあればデメリットもあります | メリットが勝る。理由は〜 / デメリットの方が痛い。なぜなら〜 |
| あまり推奨されません | やめたほうがいい |
| 注意が必要です | 具体的に何を注意すべきか書く |
| 状況に応じて最適な方法は異なります | 自分ならこうする。条件が変わるならこのあたりが分岐点 |
| 賛否が分かれるテーマです | 自分は〇〇派。理由は〜 |
| 一般的に有効だと考えられます | 有効。実際にこういうデータがある |

### パターン5: 抽象語と万能語の乱用

具体の絵が浮かばない文章。

禁止ワード:

- 「本質」「最適化」「価値を最大化」「レバレッジ」→ 何がどうなるか、動詞中心で書く
- 「非常に有効です」「大きなメリットがあります」→ 根拠の数値やファクトを添える。根拠なしの強い評価は禁止
- 「重要」「大切」「欠かせない」の同義語連打 → 1回で言い切る
- 「イノベーション」「パラダイムシフト」「エコシステム」→ 何が変わるのか、誰にとって嬉しいのかを具体的に

悪い例:
```
この銘柄は非常に魅力的で、長期的に大きなリターンが期待できます。AIという成長分野のリーダーとして、本質的な価値は高いと言えるでしょう。
```

良い例:
```
売上が前年比57%伸びていて、粗利率73%。この規模でこの成長率は異常値に近い。AIアクセラレータ市場の8割超を握っている以上、少なくとも向こう2〜3年は代替が効かない。
```

### パターン6: 不自然な比喩

AIが好む定番の比喩表現。何度も見かけると読者は気づく。

使用注意リスト:
```
羅針盤、コンパス、地図、設計図、土台、柱、
エンジン、車の両輪、潤滑油、DNA、
筋トレ、栄養、レシピ、スパイス
```

→ 比喩を使うなら、自分の体験や具体的な場面から引く。手垢のついた比喩は避ける。比喩なしで説明できるならそれが最善。

## プラットフォーム別トーン設定

### note

| 設定項目 | 値 |
|---------|------|
| 基本トーン | ですます調 + 適度な崩し |
| 一人称 | 統一する（「僕」 or 「私」）。記事内で混在させない |
| 文の長さ | 1文30〜50字を基本。たまに長文（80字超）や短文（15字以下）を混ぜてリズムを作る |
| 段落 | 3〜5行で改行。スマホ画面で圧迫感を出さない |
| 読者との距離感 | 親しみやすく。ただし馴れ馴れしくはしない |
| スタンス | 明確に取る。「自分はこう考える」を必ず含める |
| 経験談 | 1記事につき最低1箇所。自分のポジション、判断した経緯、失敗談など |
| 免責事項 | 冒頭と末尾に配置（snippets 参照） |

### X（Twitter）

| 設定項目 | 値 |
|---------|------|
| 基本トーン | 歯切れよく、断定的 |
| 文字数 | 140字を意識。超える場合はスレッド化 |
| 1投稿1メッセージ | 1つの投稿に1つの主張だけ |
| フック | 冒頭の1行で注意を引く。数字・問い・逆説が有効 |
| CTA | note記事への誘導は明確に |
| ハッシュタグ | 2〜3個まで。多すぎると広告臭 |
| 語尾 | 「だ・である」調が映えやすい。ですます調も可だが冗長に注意 |

#### X 投稿パターン例

データ型（数字で引く）:
```
S&P500、週間+2.3%。
3週連続の上昇。
テック主導の上げが続くが、RSPは+0.8%と出遅れ。
幅の狭いラリーがどこまで持つか。
```

問いかけ型（考えさせる）:
```
NVIDIAのPER 23倍を「割安」と見るか「成長鈍化の織り込み」と見るか。
答えは足元の粗利率73%をどう評価するかで変わる。
```

逆説型（意外性で引く）:
```
「分散投資が大事」と言われるけど、
Mag7だけでS&P500の3割を占めている時点で、
インデックスを買った瞬間に集中投資している。
```

## コンテンツ別トーン設定

各コンテンツタイプでのトーン微調整。基本はプラットフォーム設定に準拠し、以下で上書きする。

### 週次マーケットレポート

| 設定項目 | 値 |
|---------|------|
| トーン | 分析的。データを淡々と示しつつ、最後に主観的見解を添える |
| データ密度 | 高。数字は具体的に |
| 語りの余白 | 少なめ。情報量重視 |
| スタンス | 「来週の注目点」で方向感を示す |
| 文字数目安 | 3,000〜5,000字 |
| 禁止 | 「今週もお疲れ様でした」的な定型挨拶 |

### 個別銘柄分析

| 設定項目 | 値 |
|---------|------|
| トーン | 探究的。「一緒にこの銘柄を掘ってみよう」という姿勢 |
| データ密度 | 高。財務数値は必ず含める |
| スタンス | 強気/弱気/中立を明確に。目標株価を示すなら根拠も |
| 自分のポジション | 持っているか、検討中か、見送ったか。正直に書く |
| 文字数目安 | 4,000〜6,000字 |
| 禁止 | 「総合的に判断すると魅力的な銘柄です」のような曖昧な結論 |

### 経済指標解説

| 設定項目 | 値 |
|---------|------|
| トーン | 教育的だが説教臭くない。「知っておくと得する」くらいの温度 |
| 日常との接点 | 必ず入れる。「これが上がると、住宅ローンの金利が〜」等 |
| 歴史的文脈 | 過去の類似局面との比較を入れる |
| 文字数目安 | 2,500〜4,000字 |
| 禁止 | 「経済指標は投資判断において非常に重要な役割を果たします」的な自明の前置き |

### 投資教育

| 設定項目 | 値 |
|---------|------|
| トーン | メンター的。上から教えるのではなく、隣で一緒に考える姿勢 |
| 経験談 | 必須。失敗談があるとなお良い |
| 構成 | 抽象→具体の流れ。概念を説明した後、必ず実例を |
| 文字数目安 | 3,000〜5,000字 |
| 禁止 | 「投資の世界では」「資産形成において」等の教科書的な書き出し |

### 定量分析

| 設定項目 | 値 |
|---------|------|
| トーン | 客観的、データドリブン。ただし解釈は主観で |
| 可視化 | チャートやテーブルを積極活用 |
| 解釈の余地 | 「このデータからは〇〇と読めるが、別の見方もある」と余白を残す |
| 文字数目安 | 3,000〜5,000字 |
| 禁止 | データの羅列だけで終わる。必ず「で、これは何を意味するのか」を書く |

## ターゲット読者別設定

### 投資初心者向け

| 設定項目 | 値 |
|---------|------|
| 専門用語 | 初出時に必ず平易に説明。「PER（株価が利益の何倍かを示す指標）」 |
| 文の長さ | 短め。1文30〜40字を意識 |
| トーン | 語りかけ調、共感ベース。「初めて見ると面食らいますよね」 |
| 比喩 | 日常生活に紐付けた例え。ただしAI的な定番比喩は避ける |
| 前提知識 | ゼロを想定。略語は必ず展開 |
| 図解 | 積極活用 |

### 中級者・上級者向け

| 設定項目 | 値 |
|---------|------|
| 専門用語 | そのまま使用。必要に応じて補足 |
| データ | 具体的な数値を積極的に使用。出典明記 |
| トーン | やや硬め、分析的。ただし淡々としすぎない |
| 独自見解 | 積極的に提示。根拠付きで |
| 省略 | 基礎的な説明は省略可。「ご存知の通り」も不要、そのまま話を進める |
| 議論の余地 | 読者が自分で考えられる余白を残す |

## 人間らしさを出すテクニック

### 1. バランスを崩す（スタンスを取る）

メリットとデメリットを並べて終わり、は禁止。

- 「で、どっちがいいの？」に答える
- 自分のポジションを明かす
- 「〜だと思っている」「〜と考えている」で立場を明示
- 賛否が生まれる主張を恐れない

例:
```
悪い: NVIDIAにはメリットもデメリットもあります。投資判断は慎重に行いましょう。
良い: 自分はNVIDIAを持っている。PER 23倍でこの成長率なら、まだ安いと思っているからだ。ただし、カスタムチップの台頭は本気で警戒している。
```

### 2. 客観を崩す（主観と経験を入れる）

ファクトとデータの分析はAIに任せられる。人間が出すべきは主観と一次情報。

- 「自分はこうしている」「実際にやってみたら」
- 自分の投資判断とその理由
- 失敗談、想定外だったこと
- 偏った価値観も堂々と出す

経験談の挿入ポイント:
- 記事の導入部:「なぜこのテーマを書こうと思ったか」
- データの解釈時:「この数字を見て自分はこう感じた」
- 結論部:「自分のポートフォリオではこう対応している」

経験談がない場合の代替手法:
- データを自分で掘った過程を書く:「実際にFREDのデータを引っ張ってみたら〜」
- 判断の分岐点を示す:「もし自分が今からポジションを取るなら〜」
- 読者への率直な開示:「正直、ここは確信が持てない」

### 3. 論理を崩す（余白を作る）

完璧に構造化された文章はAIっぽい。人間の文章は少し乱れている。

- 結論に至るまでの葛藤を見せる
- 脱線を恐れない（ただし戻ってくる）
- 「あってもなくてもいい雑談」を適度に挟む
- 余韻を残す終わり方。全部説明しきらない

ただし注意:
- 週次マーケットレポートなど情報密度が求められるコンテンツでは、崩しすぎない
- 論理を崩すのは「読み物」としての記事に限定する
- 数値やファクトは正確に。崩すのは語り口であって、事実ではない

## 記号・表記ルール

### note 投稿時

| ルール | 詳細 |
|--------|------|
| マークダウン記法 | 完全禁止。`**`、`##`、`- `（箇条書き記号）、コードブロック、いずれも残さない |
| 見出し | noteのエディタ機能で設定。マークダウンの `#` は使わない |
| 太字・強調 | noteのエディタ機能で設定。`**` は使わない |
| 「」 | 引用、固有名称、会話文のみ。強調目的の「」は禁止 |
| （） | 1段落に1回まで。補足は本文に溶かす |
| ！ | 控えめに。1セクションに1回まで |
| … | 正式には「……」（二点リーダー×2）。「...」は避ける |
| 数字 | 半角。桁区切りのカンマは任意 |
| 英語 | 固有名詞は英語のまま。一般名詞は日本語に |

### X（Twitter）投稿時

| ルール | 詳細 |
|--------|------|
| マークダウン記法 | 禁止 |
| 改行 | 意味の区切りで入れる。詰め込まない |
| URL | 末尾に配置 |
| メンション | 必要時のみ |

## 投稿前チェックリスト

投稿前に上から順に確認する。

### 記号・フォーマット（必須）
- [ ] マークダウン記法（`**`、`##`、`- `、```` ）が残っていないか
- [ ] `—`（エムダッシュ）が残っていないか
- [ ] `：` + 半角スペースの組み合わせがないか
- [ ] `／` で概念を並列していないか
- [ ] 「」が多用されていないか（引用・固有名称以外）
- [ ] （）が多用されていないか（1段落1回まで）

### リズム・構成
- [ ] 同じ語尾が3回以上連続していないか
- [ ] 接続詞が3文以上連続で使われていないか
- [ ] 前置きが長すぎないか（3行以内で本題に入っているか）
- [ ] 「以下の〇つの観点から説明します」的な構造宣言がないか
- [ ] 「ステップ」「STEP」表現が残っていないか
- [ ] 「まとめると」「要するに」が2回以上出てこないか

### スタンス・人間味
- [ ] スタンスを取っているか（「自分はこう思う」が1箇所以上あるか）
- [ ] 経験談または主観的な判断が1箇所以上含まれているか
- [ ] 「一概には言えませんが」「ケースバイケース」等の逃げ表現がないか
- [ ] 抽象語だけで押し切っている段落がないか
- [ ] 根拠なしの強い評価（「非常に有効」「大きなメリット」）がないか

### 比喩・表現
- [ ] AI定番比喩（羅針盤、エンジン、土台、etc.）を使っていないか
- [ ] 文章の温度に強弱があるか（全文同じテンションになっていないか）
