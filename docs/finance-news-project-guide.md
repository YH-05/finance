# Finance News Collection プロジェクトガイド

## 概要

`Finance News Collection` は、`/collect-finance-news` コマンドで収集した金融ニュースを管理するための専用GitHub Projectです。既存の開発用プロジェクトとは独立して運用されます。

- **プロジェクト番号**: #15
- **プロジェクトURL**: https://github.com/users/yukihata-biz/projects/15

## カテゴリ別分類ルール

### 米国株 (US Stocks)
- 米国市場の個別銘柄分析
- S&P500、ナスダック構成銘柄
- 米国企業の決算・業績発表
- 例：Apple, Microsoft, Tesla等の個別銘柄分析

### 日本株 (Japan Stocks)
- 日本市場の個別銘柄分析
- 日経平均、TOPIX構成銘柄
- 日本企業の決算・業績発表
- 例：トヨタ、ソニー、キーエンス等の個別銘柄分析

### セクター分析 (Sector Analysis)
- 特定セクター（テクノロジー、金融、ヘルスケア等）の動向
- セクター別指数の分析
- 業界トレンドと競争環境
- セクターローテーション分析

### テーマ投資 (Thematic Investing)
- AI、EV、再生可能エネルギー等の成長テーマ
- テーマ別投資機会の発掘
- テーマ別ETFの動向
- 破壊的イノベーションに関する分析

### マクロ経済 (Macroeconomics)
- 経済指標（GDP、雇用統計、インフレ、PMI等）
- 中央銀行の金融政策（Fed、日銀、ECB等）
- 地政学リスクと市場への影響
- 為替・金利動向

### その他 (Misc)
- 上記カテゴリに当てはまらないニュース
- 投資教育コンテンツ
- 市場全般の話題
- 規制・法制度の変更

## カスタムフィールド

### 収集日時 (DATE)
ニュースを収集した日付。自動的に設定されます。

### 情報源 (TEXT)
RSSフィード名またはソース名。例：
- Bloomberg
- Reuters
- 日経新聞
- Wall Street Journal

### 優先度 (SINGLE_SELECT)
記事化の優先度を示します：
- **High**: 速報性が高い、読者の関心が高い、独自性がある
- **Medium**: 通常の記事化対象
- **Low**: 時間があれば記事化

### 記事化状態 (SINGLE_SELECT)
記事化の進捗状況：
- **未着手**: まだ記事化に着手していない
- **執筆中**: 記事作成中
- **完了**: 記事が完成し公開済み

## 運用フロー

### 1. ニュース収集
```bash
/collect-finance-news
```
コマンドを実行すると、RSSフィードから最新の金融ニュースを収集し、自動的にこのプロジェクトにIssueとして追加されます。

### 2. カテゴリ分類
- 収集時に自動判定されますが、必要に応じて手動で変更可能
- カテゴリフィールドを更新することで分類を調整

### 3. 優先度設定
- ニュースの重要性、時事性、読者の関心度を考慮して優先度を設定
- 高優先度のニュースから順に記事化を進める

### 4. 記事化作業
1. **未着手** → 記事化を開始する際に **執筆中** に変更
2. リサーチと執筆を実施
3. 記事公開後に **完了** に変更

### 5. アーカイブ
- 完了した記事は定期的にクローズまたはアーカイブ
- 古いニュース（30日以上経過）で未着手のものは優先度を見直し

## 自動化連携

### `/collect-finance-news` コマンド
- `finance-news-collector` エージェントが実行
- RSSフィードから新着記事を検出
- 自動的にこのプロジェクトにIssueを作成
- カテゴリ、情報源、収集日時を自動設定

### エージェント設定
`finance-news-collector` エージェントには以下の設定が必要：
```json
{
  "project_number": 15,
  "project_url": "https://github.com/users/yukihata-biz/projects/15"
}
```

## ベストプラクティス

### カテゴリ分類のコツ
- 複数カテゴリに該当する場合は、最もフォーカスしているテーマを選択
- 迷った場合は「その他」を選択し、後で再分類

### 優先度判断基準
- **High**: 24時間以内に記事化すべき速報性の高いニュース
- **Medium**: 1週間以内に記事化すべき通常のニュース
- **Low**: 時間があれば記事化する補足的なニュース

### 記事化の効率化
1. 同じカテゴリのニュースをまとめて処理
2. 優先度順にソートして上から順に着手
3. 執筆中のステータスは最小限（同時並行しすぎない）

## トラブルシューティング

### Issue が自動作成されない
- `finance-news-collector` エージェントのプロジェクト設定を確認
- RSSフィードの接続状態を確認
- ログを確認: `data/raw/rss/` ディレクトリ

### カテゴリが正しく自動判定されない
- 自動判定ロジックの改善が必要な可能性
- 手動でカテゴリを修正し、フィードバックを記録

### 古いニュースが溜まる
- 定期的に完了したIssueをクローズ
- 30日以上経過した未着手Issueは優先度を見直し
