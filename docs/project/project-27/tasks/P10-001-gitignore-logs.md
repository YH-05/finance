# P10-001: .gitignoreからlogs/*.log除外

## 概要

`logs/` ディレクトリのログファイルをGit管理対象に追加し、ワークフロー実行履歴を追跡可能にする。

## 背景

現在 `.gitignore` に `*.log` が設定されており、`logs/news-workflow-*.log` が無視されている。障害分析のためにログ履歴をGit管理したい。

## 変更内容

### 変更ファイル

| ファイル | 変更内容 |
|----------|----------|
| `.gitignore` | `*.log` を `!logs/*.log` で上書き |

### 実装詳細

```diff
# .gitignore

- *.log
+ # ログファイル（logs/以外）
+ *.log
+ !logs/*.log
```

## 受け入れ条件

- [ ] `logs/news-workflow-*.log` がGit管理対象になる
- [ ] 他のディレクトリの `*.log` は引き続き無視される
- [ ] `git status` で `logs/` 内のログファイルが表示される
