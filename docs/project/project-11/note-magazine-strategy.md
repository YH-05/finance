# noteマガジン運営戦略

**作成日**: 2026-01-14
**関連プロジェクト**: note金融コンテンツ発信強化 (#11)

## 概要

note.comでの金融市場調査・投資調査コンテンツ発信における戦略的指針を定義する。本ドキュメントは、マガジン構成、ターゲット読者層、コンテンツガイドラインを明文化し、一貫性のある高品質なコンテンツ提供を実現する。

---

## マガジン構成

financeプロジェクトでは、以下の3つのマガジンを軸としたコンテンツ発信を行う。

### 1. 週刊マーケットレポート

| 項目 | 内容 |
|------|------|
| **頻度** | 毎週月曜日 |
| **内容** | 先週の市場サマリー、今週の注目イベント、主要インデックス動向 |
| **カテゴリ** | `market_report` |
| **ターゲット** | 初心者〜中級者 |
| **価格** | 無料 |
| **文字数** | 2,000-3,000字 |

**目的**: 初心者が市場の全体像を把握できるよう、週次での振り返りと今後の展望を提供する。

**標準的な構成**:
1. エグゼクティブサマリー（200-300字）
2. 主要インデックス動向（S&P 500、NASDAQ、ダウ、日経225）
3. セクター別パフォーマンス
4. 今週の注目イベント（決算発表、経済指標発表）
5. チャート図解（2-3点）
6. まとめと見通し

### 2. 銘柄分析レポート

| 項目 | 内容 |
|------|------|
| **頻度** | 月2-4回 |
| **内容** | 個別銘柄の詳細分析、決算解説、業績トレンド |
| **カテゴリ** | `stock_analysis` |
| **ターゲット** | 中級者 |
| **価格** | 有料/無料混在 |
| **文字数** | 3,000-5,000字 |

**目的**: 個別銘柄への深掘り分析を通じ、企業価値評価の視点を提供する。

**標準的な構成**:
1. 企業概要（事業内容、市場ポジション）
2. 最新決算サマリー（売上、利益、EPS）
3. 財務指標分析（PER、ROE、営業利益率等）
4. テクニカル分析（チャート、移動平均線）
5. リスク要因
6. 総合評価（ただし投資助言は避ける）

### 3. 投資戦略・教育

| 項目 | 内容 |
|------|------|
| **頻度** | 月2回 |
| **内容** | クオンツ分析、投資手法解説、経済指標の読み方 |
| **カテゴリ** | `quant_analysis`, `investment_education`, `economic_indicators` |
| **ターゲット** | 上級者 |
| **価格** | 有料 |
| **文字数** | 2,500-6,000字 |

**目的**: 定量分析や高度な投資戦略を解説し、読者の投資スキル向上を支援する。

**主なトピック例**:
- ファクター投資（バリュー、グロース、モメンタム）
- ポートフォリオ理論（分散投資、リバランス）
- 経済指標の解説（雇用統計、GDP、インフレ率）
- リスク管理手法（ボラティリティ、最大ドローダウン）

## ターゲット読者層

読者を3つのレベルに分類し、それぞれに適したコンテンツを提供する。

### 初心者（投資未経験〜1年）

**特徴**:
- 投資用語に不慣れ
- 市場の仕組みを学びたい
- 「何から始めればいいか分からない」

**ニーズ**:
- 基礎知識の習得
- 専門用語の解説
- 市場全体の動きの理解

**対応カテゴリ**:
- `market_report`（週刊マーケットレポート）
- `investment_education`（投資教育コンテンツ）

**コンテンツの工夫**:
- 専門用語は必ず初出時に解説
- 図解を多用し、視覚的に理解しやすく
- 「今週のポイント3つ」など要約セクションを設ける

### 中級者（投資1-5年）

**特徴**:
- 基本的な投資経験あり
- 個別銘柄の選定に興味
- 決算情報の読み方を学びたい

**ニーズ**:
- 市場動向の深い理解
- 銘柄選定のヒント
- 決算解説・業績分析

**対応カテゴリ**:
- `market_report`（週刊マーケットレポート）
- `stock_analysis`（銘柄分析レポート）
- `economic_indicators`（経済指標解説）

**コンテンツの工夫**:
- 財務指標の比較表を掲載
- 競合他社との比較分析
- 「注目ポイント」を明示

### 上級者（投資5年以上）

**特徴**:
- 投資戦略を自分で構築できる
- 定量的な分析手法に関心
- ファクター投資やクオンツに興味

**ニーズ**:
- 戦略の検証・バックテスト結果
- 高度な定量分析
- 最新の投資理論

**対応カテゴリ**:
- `quant_analysis`（クオンツ分析）
- `investment_education`（高度な投資戦略）

**コンテンツの工夫**:
- 統計データ・シャープレシオ等を掲載
- Pythonコードやバックテスト結果を共有
- 論文や研究レポートの引用

## コンテンツガイドライン

### 文字数目安

| カテゴリ | 文字数 |
|---------|--------|
| 市場レポート (`market_report`) | 2,000-3,000字 |
| 銘柄分析 (`stock_analysis`) | 3,000-5,000字 |
| 投資教育 (`investment_education`) | 2,500-4,000字 |
| 経済指標 (`economic_indicators`) | 2,500-4,000字 |
| クオンツ分析 (`quant_analysis`) | 4,000-6,000字 |

### トーン&スタイル

| 項目 | 規則 |
|------|------|
| **文体** | 敬体（です・ます調） |
| **専門用語** | 初出時に必ず解説を付ける |
| **数値表現** | 統一（例: 「1,000円」「1.5%」「10億ドル」） |
| **引用** | 出典を明記する（例: 「Bloomberg, 2026-01-10」） |
| **バイアス回避** | 「私の意見では」等は避け、データに基づく記述を優先 |

### 必須要素

すべての記事に以下を含める：

1. **サムネイル画像** - 1200x630px推奨、記事内容を象徴する図やチャート
2. **目次** - 3,000字以上の記事には必須
3. **チャート/図表** - 最低1つ以上（できれば2-3つ）
4. **データソース** - 使用したデータの出典（例: yfinance, FRED, SEC EDGAR）
5. **免責事項** - 記事末尾に必ず記載（`snippets/disclaimer.md`を使用）

### 免責事項テンプレート

すべての記事の末尾に以下を記載する：

```markdown
---

## 免責事項

本記事は公開されている情報を基に作成しており、特定の投資助言や推奨を意図するものではありません。投資判断は自己責任で行ってください。記事の内容は執筆時点の情報であり、将来の市場動向を保証するものではありません。
```

### 禁止表現

以下の表現は金融商品取引法における投資助言に該当する可能性があるため、使用を禁止する：

| 禁止表現 | 代替表現例 |
|---------|-----------|
| 「買うべき」「売るべき」 | 「注目される」「検討に値する」 |
| 「絶対」「必ず」「確実に」 | 「一般的に」「多くの場合」「傾向として」 |
| 「おすすめ銘柄」 | 「注目銘柄」「市場で話題の銘柄」 |
| 「今が買い時」 | 「現在の株価水準は〜」 |

## 記事作成ワークフロー

### 週刊マーケットレポートの標準フロー

```
1. /new-finance-article "S&P500週間レビュー" --category market_report
2. /finance-research（市場データ収集、Web検索、Wiki検索）
3. [HF] リサーチ結果確認（decisions.json）
4. /finance-edit --mode quick（初稿作成→批評→修正）
5. [HF] 最終レビュー
6. note.comへ公開
```

### 銘柄分析レポートの標準フロー

```
1. /new-finance-article "AAPL Q1決算分析" --category stock_analysis
2. /finance-research（yfinance、SEC EDGAR、Web検索）
3. [HF] リサーチ結果確認
4. /finance-edit --mode full（詳細な批評プロセス）
5. [HF] 最終レビュー
6. note.comへ公開
```

## データソースと活用方針

### 主要データソース

| データソース | 用途 | 対応パッケージ |
|------------|------|--------------|
| **yfinance** | 株価、為替、ETF価格 | `market_analysis` |
| **FRED** | 米国経済指標 | `market_analysis` |
| **SEC EDGAR** | 企業決算、Form 4 | MCP経由（Phase 1で実装予定） |
| **Web検索** | ニュース、市場センチメント | `finance-web` エージェント |
| **Wikipedia** | 企業・経済用語の背景情報 | `finance-wiki` エージェント |

### 標準的な分析項目

#### 週刊マーケットレポート

- **主要インデックス**: S&P 500, NASDAQ, ダウ, 日経225
- **セクターETF**: 11セクターの週間リターン
- **スタイルファクター**: グロースvs.バリュー（VUG vs VTV）、大型vs.小型（SPY vs IWM）
- **Magnificent 7**: 7銘柄の週間パフォーマンス
- **マクロ指標**: 金利、為替、VIX

#### 銘柄分析レポート

- **株価データ**: 直近1年間のチャート、移動平均線
- **財務指標**: PER, PBR, ROE, 営業利益率
- **決算トレンド**: 直近4四半期の売上・利益推移
- **テクニカル指標**: RSI, MACD（必要に応じて）

## 品質チェックリスト

記事公開前に以下を確認する：

### 内容面
- [ ] タイトルは明確で検索されやすいか
- [ ] 専門用語は初出時に解説されているか
- [ ] 免責事項が記載されているか
- [ ] 禁止表現（「買うべき」等）が含まれていないか
- [ ] データソースが明記されているか

### 視覚面
- [ ] サムネイル画像が設定されているか
- [ ] チャート/図表が最低1つ含まれているか
- [ ] 目次が設定されているか（3,000字以上の場合）
- [ ] 見出しレベルが適切か（H2→H3の階層構造）

### 技術面
- [ ] リンクが正しく機能するか
- [ ] 画像が表示されるか
- [ ] 文字数が目安範囲内か
