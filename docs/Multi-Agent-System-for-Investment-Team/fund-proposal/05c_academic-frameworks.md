# V-D. 学術的フレームワークとの接続 — 6つの学術的根拠

> アプローチD: 投資の意思決定品質を評価する既存の学術研究とMASの設計を接続し、方法論の妥当性を権威で担保する。

---

## D-1. Kahneman「Noise」— 判断のばらつき削減

2021年の著書「Noise: A Flaw in Human Judgment」の核心テーマ。

**主張**: 人間の判断における最大の問題は「バイアス」ではなく「ノイズ」（同じ条件で異なる判断をしてしまうばらつき）。

**MAS との接続**:
- 構造化されたプロセスにより、判断のノイズを体系的に削減
- A-3 の一貫性スコアで、ノイズ削減を定量的に実証可能
- 「同じ銘柄を3回分析して、結果のばらつきを測定する」テストが可能

**投資家への訴求**: 「ノーベル経済学賞受賞者 Kahneman が指摘した"判断のノイズ"を、構造的に排除する投資プロセスです」

## D-2. Tetlock「Superforecasting」— 予測精度の分解

「超予測力」で示された、優れた予測者の特徴：

| 特徴 | MAS での実装 |
|------|------------|
| 問題を分解する | 4アナリストが独立分析 |
| 確率的に考える | 確信度スコア（10-90%） |
| ベースレートを重視する | ファクタースクリーニング |
| アップデートする | Postmortem で過去の失敗を分析 |
| 反対意見を積極的に取り入れる | Bull/Bear ディベート |

**投資家への訴求**: 「Tetlock の研究が示す"超予測者"の5つの特徴を、投資プロセスに構造的に組み込んでいます」

## D-3. Mauboussin「The Success Equation」— スキルと運の分離

Michael Mauboussin のフレームワーク：投資成果 = スキル + 運。

**重要な示唆**: 短期的にはスキルが高くても運で負けることがある。だからこそ**プロセスの質を評価するべき**。

**MAS との接続**:
- リターンではなくプロセスを評価するのは、Mauboussin の「スキルと運の分離」に直結
- A-5 の予測分解精度は、まさに「スキル」の測定
- 十分な期間をとれば、スキルが高いプロセスは統計的にアウトパフォームする

**投資家への訴求**: 「投資成果は"スキル"と"運"の合計です。運は制御できませんが、スキル（分析の質）は測定・改善できます。当ファンドはスキルの質を継続的に測定し公開します」

## D-4. SDG（Strategic Decisions Group）の Decision Quality フレームワーク

スタンフォード大学発の意思決定品質フレームワーク。6つの要素で判断の質を評価：

| 要素 | 定義 | MAS での実装 |
|------|------|------------|
| Appropriate Frame | 正しい問題設定 | スクリーニングで投資候補を絞り込み |
| Creative Alternatives | 複数の選択肢 | Bull/Bear が異なるシナリオを提示 |
| Relevant Information | 適切な情報 | 4アナリストが独立にデータ収集 |
| Clear Values | 明確な価値基準 | dogma.md で投資哲学を明文化 |
| Sound Reasoning | 健全な推論 | 12ルールによる因果推論チェック |
| Commitment to Action | 実行へのコミットメント | 確信度スコアに基づくウェイト配分 |

**投資家への訴求**: 「スタンフォード大学の Decision Quality フレームワーク（6要素）を投資プロセスに完全実装しています」

## D-5. 行動ファイナンスの「デバイアシング」

行動ファイナンスで知られるバイアスと、MAS の構造的排除方法の対応：

| バイアス | 人間のファンドでの影響 | MAS の構造的排除方法 |
|---------|--------------------|--------------------|
| 確証バイアス | 自分の仮説に合う情報だけ集める | Bull/Bear が反対の立場を強制的に取る |
| アンカリング | 前回の判断に引きずられる | 毎回ゼロベース分析（Analyst Y の最大の関心事） |
| 損失回避 | 負けポジションを切れない | 確信度スコアが閾値以下なら機械的に除外 |
| ハーディング | 市場コンセンサスに追随 | 独立分析、ティッカー匿名化 |
| 過信 | 自分の分析を過大評価 | T8 自己批判プロセス |
| 可用性バイアス | 最近の記憶に引きずられる | Point-in-Time データ管理 |

**投資家への訴求**: 「行動ファイナンスが特定した6つの主要バイアスを、プロセス設計によって構造的に排除しています」
