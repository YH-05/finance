意見が対立した場合の最終判断プロセスは、システムの種類によって主に**「全会一致を目指すパターン」**と**「上位の管理者が決定するパターン」**の2つに分かれます。

### 1. 全会一致型（AlphaAgentsの例）

エージェント全員が納得するまで議論を続けます。

- **プロセス**: ラウンドロビン方式（持ち回り）で発言し、他者の指摘を受けて自分の分析を修正します。これを**合意（Consensus）に至るまで**繰り返します。
- **具体例**: Zscaler株の事例では、当初「買い」を主張していたValuation Agentが、他のエージェントから財務リスクを指摘され、最終的に「売り」に意見を変更し、全員一致で「売り」と決まりました[Figure 5]。

### 2. マネージャー決定型（TradingAgents / HedgeAgentsの例）

議論の内容を踏まえて、最終決定権を持つ**「ファンドマネージャー」役のエージェント**が判断を下します。

- **プロセス**: アナリストやリスク管理チーム（強気派・弱気派・慎重派など）が議論した内容を、上位のマネージャーAI（例：Fund Manager）がレビューし、最終的な取引執行（Execution）を承認します[Figure 1, 504]。
- **具体例**: Apple株の事例では、リスク管理チーム内で意見が分かれましたが、マネージャーが「割高感やインサイダーの売りという懸念はあるものの、ファンダメンタルズの強さが勝る」と総合的に判断し、ストップロス（損切り）設定などの条件付きで「買い」を決定しました。

### 補足：人間の介入

どちらのシステムでも、エージェント間の議論ログはすべて記録されており、必要に応じて**人間がレビューして最終決定を覆す**ことも想定されています。

このように、「AI同士に納得いくまで話し合わせる」か、「AIの上司が決断する」かという違いがあります。
