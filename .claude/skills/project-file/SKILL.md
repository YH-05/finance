# プロジェクトファイルスキル

このスキルは、パッケージ開発の計画を記述するプロジェクトファイル（project.md）を作成・編集するためのガイドです。

## プロジェクトファイルとは

プロジェクトファイルは、パッケージの実装計画を定義するドキュメントです。

- **目的**: パッケージで実装する機能とマイルストーンを定義
- **配置場所**: `src/<package_name>/docs/project.md`
- **作成タイミング**: `/new-package` コマンドで自動生成、または手動で作成

## 役割と位置づけ

```
project.md（実装計画）
       │
       ▼
/new-project で LRD・設計ドキュメント作成
       │
       ▼
/issue → feature-implementer で機能実装
```

- **project.md**: 何を作るかの計画（マイルストーン、機能一覧）
- **LRD**: 詳細な要求定義（ユーザーストーリー、受け入れ条件）
- **設計ドキュメント**: 技術的な設計（アーキテクチャ、機能設計）

## テンプレートの参照

プロジェクトファイルを作成する際は、次のテンプレートを使用してください: ./template.md

## プロジェクトファイルの主要セクション

### 1. 概要

パッケージの目的と背景を簡潔に記述します。

```markdown
## 概要

このパッケージは [目的] を提供します。

**背景**:
- [背景1]
- [背景2]

**解決する課題**:
- [課題1]
- [課題2]
```

### 2. 実装計画

マイルストーンと機能を階層的に定義します。

```markdown
## 実装計画

### マイルストーン 1: [名前] (期限: YYYY-MM-DD)

#### 機能 1.1: [機能名]
- 優先度: P0
- 説明: [詳細説明]
- 受け入れ条件:
  - [ ] 条件1
  - [ ] 条件2
```

### 3. 技術的考慮事項

実装に影響する技術的な制約や選択を記述します。

```markdown
## 技術的考慮事項

- **アーキテクチャ**: [選択した設計パターン]
- **依存関係**: [外部ライブラリの選定理由]
- **制約事項**: [既知の制限]
```

## 優先度の定義

機能には以下の優先度を設定します:

- **P0(必須)**: MVI に含める機能。これがないとパッケージとして成立しない
- **P1(重要)**: 初期リリース後すぐに追加すべき機能
- **P2(できれば)**: 将来的に追加を検討する機能

## 作成のポイント

### 1. 具体的な機能定義

```markdown
# 悪い例
- データ処理機能

# 良い例
#### 機能 1.1: JSON パーサー
- 優先度: P0
- 説明: JSON 文字列を Python dict に変換する
- 受け入れ条件:
  - [ ] 有効な JSON を dict に変換できる
  - [ ] 無効な JSON に対して明確なエラーメッセージを返す
```

### 2. 測定可能な受け入れ条件

すべての受け入れ条件は、完了したかどうかを判断できる形式で記述します。

```markdown
# 悪い例
- [ ] 速い
- [ ] 使いやすい

# 良い例
- [ ] 1000 要素のリスト処理が 10ms 以内
- [ ] README のサンプルコードが 5 分以内で実行可能
```

### 3. マイルストーンの粒度

マイルストーンは 1-2 週間で完了できる規模を目安にします。

## 更新のタイミング

プロジェクトファイルは以下のタイミングで更新します:

1. **新機能の追加時**: マイルストーンに新しい機能を追加
2. **スコープ変更時**: 優先度の変更やマイルストーンの再編成
3. **完了時**: 完了した機能をマーク（`- [x]`）

## 関連コマンド

- `/new-package <name>`: 新規パッケージと project.md を作成
- `/new-project @src/<pkg>/docs/project.md`: LRD・設計ドキュメント作成
- `/issue @src/<pkg>/docs/project.md`: Issue管理・タスク分解
