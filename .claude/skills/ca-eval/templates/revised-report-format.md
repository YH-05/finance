# revised-report.md フォーマットテンプレート

T8（AI批判プロセス）で `draft-report.md` を批判的に検証し、修正を反映した最終版レポートのフォーマット。
Lead（ca-eval-lead）が直接実行する。Yに渡す最終版。

## draft-report.md との差分

revised-report.md は draft-report.md をベースに以下の修正を適用する:

| 修正要素 | 説明 |
|---------|------|
| ヘッダー変更 | `バージョン: revised_v1（T8 AI批判プロセス適用済み）` + `T8適用` 行を追加 |
| `[⬇️ T8修正]` アノテーション | confidence が変更された主張のセクションヘッダーに付与 |
| `[注N] T8批判` セクション | 各主張内に批判内容を blockquote で追加 |
| T8 AI批判プロセスの総括 | セクション4に systematic_issues を追加 |
| T8修正サマリー | 末尾に修正一覧表を追加 |
| フッター変更 | ドラフト注記を最終版注記に変更 |

## プレースホルダー

draft-report-format.md のプレースホルダーに加えて:

| プレースホルダー | 説明 |
|----------------|------|
| `[old_confidence]` | T8修正前の確信度 |
| `[new_confidence]` | T8修正後の確信度 |
| `[critique_type]` | 批判タイプ（overconfidence / underconfidence / reasoning_gap / kb_misalignment） |
| `[severity]` | 重大度（critical / minor） |
| `[critical_issues]` | critical指摘の件数 |
| `[minor_issues]` | minor指摘の件数 |

## T8修正箇所の表示パターン

### パターン1: confidence 変更あり

```markdown
#### AI評価: おおむね納得（70%） [⬇️ T8修正: 90% → 70%]

構造的優位性（ルール6）と業界構造の合致（ルール11）が明確であり、...
<u>ただし純粋競合との配送効率の具体的比較が不足</u>しており、
KB3実績（ORLY#2, #5）では90%はルール7の具体的比較が揃った場合のみ付与されている。70%が妥当。

**T8批判**: KB3実績と照合した結果、ルール11の合致のみでは90%は過剰。
ルール7（純粋競合比較）が不足する場合は70%が適切（COST#1と同等）。
```

### パターン2: confidence 維持 + 条件付きアノテーション

```markdown
#### AI評価: おおむね納得（70%） [⬇️ T8修正: 大型化リスクのアノテーション追加]

[評価文]

> **[注N] T8批判 (critical): 大型化リスクのCAGR confidence未反映**
>
> [批判の詳細]
>
> **評価スケールは10%刻みのため、CAGR confidence 70%を維持しつつ以下の条件付きアノテーションを追加する:**
> - **現状維持（70%）**: [維持の根拠]
> - **リスク顕在化時（50%に低下）**: [低下条件]
```
