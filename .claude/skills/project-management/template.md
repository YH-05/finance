# Project.md テンプレート

このドキュメントには、project.md の2つのテンプレートが含まれています。

## テンプレート1: パッケージ開発用

`/new-package` で作成されるパッケージの開発計画用テンプレートです。

```markdown

## 概要

[パッケージの目的と背景を記載]

**背景**:
- [このパッケージを作る理由1]
- [このパッケージを作る理由2]

**解決する課題**:
- [課題1]
- [課題2]

**提供する価値**:
- [価値1]
- [価値2]

## 実装計画

### マイルストーン 1: 基本機能 (期限: YYYY-MM-DD)

このマイルストーンでは、パッケージの基本機能を実装します。

#### 機能 1.1: [機能名]
- 優先度: P0
- 説明: [詳細説明]
- 受け入れ条件:
  - [ ] [測定可能な条件1]
  - [ ] [測定可能な条件2]

#### 機能 1.2: [機能名]
- 優先度: P1
- 説明: [詳細説明]
- 受け入れ条件:
  - [ ] [測定可能な条件1]
  - [ ] [測定可能な条件2]

### マイルストーン 2: [名前] (期限: YYYY-MM-DD)

[マイルストーンの概要]

#### 機能 2.1: [機能名]
- 優先度: P1
- 説明: [詳細説明]
- 受け入れ条件:
  - [ ] [測定可能な条件1]

### マイルストーン 3: [名前] (期限: YYYY-MM-DD)

[マイルストーンの概要]

#### 機能 3.1: [機能名]
- 優先度: P2
- 説明: [詳細説明]
- 受け入れ条件:
  - [ ] [測定可能な条件1]

## 技術的考慮事項

- [設計パターンや構造に関する考慮事項]
```

## 優先度の定義

機能には以下の優先度を設定します:

| 優先度 | 定義 | 説明 |
|--------|------|------|
| **P0(必須)** | MVI に含める機能 | これがないとパッケージとして成立しない |
| **P1(重要)** | 初期リリース後すぐに追加 | 主要なユースケースに必要 |
| **P2(できれば)** | 将来的に追加を検討 | あると便利だが必須ではない |

## ステータスの定義

タスク/プロジェクトのステータス:

| ステータス | 説明 |
|-----------|------|
| **計画中** | 要件定義・設計中 |
| **進行中** | 実装中（一部タスクが In Progress） |
| **完了** | 全タスク完了、プロジェクト終了 |

タスクのステータス:

| ステータス | 説明 |
|-----------|------|
| **todo** | 未着手 |
| **in_progress** | 実装中 |
| **done** | 完了 |

## 受け入れ条件の書き方

### 悪い例

```markdown
- [ ] 速い
- [ ] 使いやすい
- [ ] テストを書く
```

### 良い例

```markdown
- [ ] 1000 要素のリスト処理が 10ms 以内
- [ ] README のサンプルコードが 5 分以内で実行可能
- [ ] 全公開関数にユニットテストを追加（カバレッジ80%以上）
- [ ] make check-all が成功する
```

## Issue 本文テンプレート

タスクを GitHub Issue として作成する際のテンプレート:

```markdown
## 概要

[タスクの概要を1-2文で記載]

## 詳細

[詳細な説明]

- 背景: [なぜこのタスクが必要か]
- 目的: [何を達成したいか]

## 受け入れ条件

- [ ] [測定可能な条件1]
- [ ] [測定可能な条件2]
- [ ] [測定可能な条件3]

## 技術的な注意点

[実装時に考慮すべき点]

## 関連

- 計画書: docs/project/{slug}.md
- GitHub Project: [#{project_number}]({project_url})
- 依存: #{依存するIssue番号}
- ブロック: #{ブロックするIssue番号}
```

## プロジェクト完了時の更新

全 Issue が完了した場合、以下を更新:

```markdown
# 更新前
**ステータス**: 計画中
**作成日**: 2026-01-15

# 更新後
**ステータス**: 完了
**作成日**: 2026-01-15
**完了日**: 2026-01-22
```

タスク一覧セクションも更新:

```markdown
# 更新前
- [ ] タスク1
  - Issue: [#123](URL)
  - ステータス: todo

# 更新後
- [x] タスク1
  - Issue: [#123](URL)
  - ステータス: done
  - 完了日: 2026-01-20
```

## Issue テーブル形式（オプション）

Issue 一覧をテーブル形式で管理する場合:

```markdown
## GitHub Issues

| # | タイトル | ラベル | Phase | ステータス |
|---|---------|--------|-------|----------|
| [#147](URL) | パッケージ調査 | enhancement | 1 | Done |
| [#148](URL) | 設計実装 | enhancement | 2 | In Progress |
| [#149](URL) | テスト追加 | test | 3 | Todo |

**全Issue完了**: 2026-01-22
```
