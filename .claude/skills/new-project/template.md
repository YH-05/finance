# 軽量プロジェクト テンプレート

## project.md テンプレート

以下のテンプレートを使用して `docs/project/{slug}.md` を作成します。

```markdown
# {プロジェクト名}

**作成日**: {今日の日付}
**ステータス**: 計画中
**GitHub Project**: [#{project_number}]({project_url})

## 背景と目的

### 背景

- **種類**: {質問1の回答}
- **課題**: {質問2の回答}
- **期間**: {質問3の回答}

### 目的

{質問4の回答}

## スコープ

### 含むもの

- **変更範囲**: {質問7の回答}
- **影響ディレクトリ**: {質問8の回答}

### 含まないもの

{質問9の回答}

## 成果物

| 種類 | 名前 | 説明 |
| ---- | ---- | ---- |
| {質問5の回答} | {成果物名} | {説明} |

## 成功基準

{質問6の回答をチェックリスト形式に変換}

- [ ] {基準1}
- [ ] {基準2}

## 技術的考慮事項

### 実装アプローチ

{質問10の回答}

### 依存関係

{質問11の回答}

### テスト要件

{質問12の回答}

## タスク一覧

### 準備

- [ ] 関連コードの調査
  - Issue: [#{番号}]({URL})
  - ステータス: todo

### 実装

- [ ] {タスク1}
  - Issue: [#{番号}]({URL})
  - ステータス: todo
- [ ] {タスク2}
  - Issue: [#{番号}]({URL})
  - ステータス: todo

### テスト・検証

- [ ] テストの作成
  - Issue: [#{番号}]({URL})
  - ステータス: todo
- [ ] make check-all の実行
  - Issue: [#{番号}]({URL})
  - ステータス: todo

### ドキュメント

- [ ] README/ドキュメントの更新（必要な場合）
  - Issue: [#{番号}]({URL})
  - ステータス: todo

---

**最終更新**: {今日の日付}
```

## Issue 本文テンプレート

```markdown
## 概要

{タスクの概要を1-2文で記載}

## 詳細

{詳細な説明}

- **背景**: {なぜこのタスクが必要か}
- **目的**: {何を達成したいか}

## 受け入れ条件

- [ ] {測定可能な条件1}
- [ ] {測定可能な条件2}
- [ ] {測定可能な条件3}

## 技術的な注意点

{実装時に考慮すべき点}

## 関連

- 計画書: docs/project/{slug}.md
- GitHub Project: [#{project_number}]({project_url})
```

## slug 変換ルール

プロジェクト名を URL-safe な slug に変換：

| 入力 | 出力 |
|------|------|
| `CI/CD改善` | `ci-cd-improvement` |
| `エージェント開発` | `agent-development` |
| `New Feature` | `new-feature` |

変換手順：
1. 日本語をローマ字/英語に変換（可能な範囲で）
2. スペース、スラッシュ → ハイフン
3. 小文字に統一
4. 連続するハイフンを単一に
5. 先頭・末尾のハイフンを削除
