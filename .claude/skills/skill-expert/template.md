## テンプレート使用時の注意

<!-- このセクションは実際のスキルでは削除 -->

このテンプレートを使用する際：

1. **フロントマターを適切に設定**
   - `name`: ディレクトリ名と一致（kebab-case）
   - `description`: トリガーキーワードを含む簡潔な説明
   - `allowed-tools`: 必要最小限のツールのみ指定

2. **各セクションをカスタマイズ**
   - 不要なセクションは削除
   - 必要に応じてセクションを追加
   - 具体的な例に置き換え

3. **実用的な内容に**
   - 抽象的な説明を避ける
   - 具体的な例を3つ以上含める
   - 実際に使用可能な内容に

4. **リソースファイルの作成**
   - `guide.md`: 詳細なガイドが必要な場合
   - `template.md`: 出力テンプレートが必要な場合
   - `examples/`: 複数の例が必要な場合

5. **検証を忘れずに**
   - 品質基準のチェックリストを確認
   - フロントマターが正しく設定されているか確認
   - 実際に使用してテスト

6. **このセクション（テンプレート使用時の注意）を削除**
   - 実際のスキルには不要

## フロントマターのフィールド説明

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `name` | ✓ | スキル名（kebab-case、ディレクトリ名と一致） |
| `description` | ✓ | Skill tool に表示される説明（トリガーキーワード含む） |
| `allowed-tools` | △ | 許可するツールのリスト（省略時は全ツール利用可） |

### description の書き方

**良い例**:
```yaml
description: |
  コーディング規約を確認・適用するためのガイドとテンプレート。
  コード品質、命名規則、型ヒントに関する質問時に使用。
```

**悪い例**:
```yaml
description: プログラミングのスキル  # 曖昧でトリガーキーワードがない
```

### allowed-tools の有効な値

- `Read`: ファイル読み取り
- `Write`: ファイル書き込み
- `Edit`: ファイル編集
- `Glob`: パターンマッチング
- `Grep`: 正規表現検索
- `Bash`: コマンド実行
- `Task`: サブエージェント起動
- `AskUserQuestion`: ユーザーへの質問
- `WebFetch`: Web取得
- `WebSearch`: Web検索
- `TodoWrite`: タスク管理
