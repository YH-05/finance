# Rules - 共通ルール定義

このディレクトリには、Claude Code のサブエージェント、コマンド、スキルが共通で参照する基本ルールを集約しています。

## 目的

- **一貫性の確保**: 全てのエージェント・コマンド・スキルが同じルールを認識
- **保守性の向上**: ルール変更を一箇所で管理
- **認識の統一**: プロジェクト全体での共通理解を促進

## ファイル構成

| ファイル | 説明 |
|---------|------|
| `coding-standards.md` | コーディング規約（型ヒント、命名規則、関数設計など） |
| `development-process.md` | 開発プロセス（Git運用、PR/Issue規則、自動化など） |
| `testing-strategy.md` | テスト戦略（TDD、テスト種別、命名規則など） |
| `git-rules.md` | Git運用ルール（ブランチ戦略、コミットメッセージ規約） |
| `common-instructions.md` | 共通指示（ロギング、エラーハンドリング、環境変数など） |
| `evidence-based.md` | エビデンスベース開発（禁止語、推奨語、証拠要件など） |
| `subagent-data-passing.md` | **🚨 サブエージェントへのデータ渡しルール（完全なデータ構造必須）** |

## 使用方法

### エージェント・スキル・コマンドでの参照

プロンプト内で以下のように参照してください：

```markdown
参照ルール:
- @.claude/rules/coding-standards.md
- @.claude/rules/testing-strategy.md
```

### CLAUDE.md との関係

`CLAUDE.md` はプロジェクトの概要と主要なルールを記載しています。このディレクトリのファイルは、`CLAUDE.md` の詳細版として機能し、エージェント等が必要に応じて個別のルールを参照できます。

## 更新ルール

- ルール変更時は該当ファイルを更新
- 大きな変更は PR でレビュー
- 変更後は関連するエージェント・スキル・コマンドの動作を確認
