# examples/ ディレクトリの使い方

このディレクトリには、スキルの使用例やパターン集を格納します。

## 目的

- スキルの具体的な使用例を提供
- 複数のパターンやバリエーションを整理
- 実際のユースケースに基づいたリファレンスを提供

## ディレクトリ構造

```
examples/
├── README.md           # このファイル（必須）
├── basic-usage.md      # 基本的な使用例
├── advanced-usage.md   # 高度な使用例
├── pattern-1.md        # パターン1
├── pattern-2.md        # パターン2
└── ...
```

## ファイル命名規則

| パターン | 説明 | 例 |
|----------|------|-----|
| `basic-*.md` | 基本的な使用例 | `basic-usage.md` |
| `advanced-*.md` | 高度な使用例 | `advanced-patterns.md` |
| `pattern-*.md` | 特定のパターン | `pattern-error-handling.md` |
| `example-*.md` | 具体的な例 | `example-api-design.md` |
| `template-*.md` | サブテンプレート | `template-header.md` |

## 例ファイルの標準構造

各例ファイルは以下の構造に従うことを推奨：

```markdown
# [例のタイトル]

## 概要

[この例が示す内容の簡潔な説明]

## 前提条件

- [前提条件1]
- [前提条件2]

## 例

### シナリオ

[状況の説明]

### 入力

\`\`\`
[入力例]
\`\`\`

### 処理

1. [ステップ1]
2. [ステップ2]

### 出力

\`\`\`
[出力例]
\`\`\`

## 解説

[なぜこのパターンを使用するか、注意点など]

## 関連

- [関連する例へのリンク]
```

## 使用例

### 例1: 基本的な使用例ファイル

`examples/basic-usage.md`:

```markdown
# 基本的な使用例

## 概要

このスキルの最も基本的な使用パターンを示します。

## 例

### シナリオ

新規プロジェクトでこのスキルを初めて使用する場合。

### 入力

...

### 出力

...
```

### 例2: パターン別ファイル

複数のパターンがある場合は、パターンごとにファイルを分けます：

```
examples/
├── README.md
├── pattern-simple.md      # シンプルパターン
├── pattern-standard.md    # 標準パターン
└── pattern-advanced.md    # 高度なパターン
```

各パターンファイルには、そのパターンの詳細な説明と例を含めます。

## SKILL.md での参照

SKILL.md のリソースセクションで examples/ を参照する例：

```markdown
## リソース

### ./examples/

使用例・パターン集:
- `basic-usage.md`: 基本的な使用例
- `pattern-simple.md`: シンプルパターン
- `pattern-advanced.md`: 高度なパターン
```

## ベストプラクティス

### DO（推奨）

- 各例に明確な目的を持たせる
- 実際のユースケースに基づいた例を作成する
- 入力と出力を具体的に示す
- 例の選択基準を README.md に記載する

### DON'T（避けるべき）

- 抽象的すぎる例
- 重複した内容の例
- コンテキストのない例
- メンテナンスされていない例

## メンテナンス

examples/ ディレクトリを追加・更新した場合：

1. この README.md を更新する
2. SKILL.md のリソースセクションを更新する
3. 古い例を削除または更新する
