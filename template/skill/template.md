## 変数の説明

| 変数名 | 説明 | 必須 | デフォルト値 |
|--------|------|------|-------------|
| `TITLE` | ドキュメントのタイトル | ✅ | なし |
| `DESCRIPTION` | 概要の説明文 | ✅ | なし |
| `SECTION_1_CONTENT` | セクション1の内容 | ✅ | なし |
| `SUBSECTION_2_1_CONTENT` | サブセクション2.1の内容 | ❌ | "（未設定）" |
| `SUBSECTION_2_2_CONTENT` | サブセクション2.2の内容 | ❌ | "（未設定）" |
| `ROW_X_COL_Y` | テーブルの各セル | ❌ | "-" |
| `CONCLUSION` | 結論の内容 | ✅ | なし |

## 使用例

### 例1: 基本的な使用

**入力値**:
```yaml
TITLE: "プロジェクト概要"
DESCRIPTION: "このプロジェクトは〜を目的としています"
SECTION_1_CONTENT: "主要な機能について説明します"
CONCLUSION: "以上がプロジェクトの概要です"
```

```markdown

## セクション1

主要な機能について説明します

### サブセクション2.1

（未設定）

### サブセクション2.2

（未設定）

## セクション3

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| - | - | - |
| - | - | - |

## 結論

以上がプロジェクトの概要です
```

## カスタマイズガイド

### セクションの追加

テンプレートにセクションを追加する場合：

```markdown
## 新しいセクション

{{NEW_SECTION_CONTENT}}
```

変数の説明テーブルに追加：

| 変数名 | 説明 | 必須 | デフォルト値 |
|--------|------|------|-------------|
| `NEW_SECTION_CONTENT` | 新しいセクションの内容 | ✅/❌ | なし/"デフォルト" |

### 条件付きセクション

オプションのセクションは、変数が設定されている場合のみ出力：

```markdown
{{#if OPTIONAL_SECTION}}
## オプションセクション

{{OPTIONAL_SECTION}}
{{/if}}
```

### 繰り返しセクション

リストの各要素に対して繰り返し：

```markdown
{{#each ITEMS}}
### {{name}}

{{description}}

{{/each}}
```

## テンプレート使用時の注意

<!-- このセクションは実際のテンプレートでは削除すること -->

このテンプレートファイルを使用する際：

1. **目的に応じてカスタマイズ**
   - 不要なセクションは削除
   - 必要なセクションを追加
   - 変数名を適切に変更

2. **変数の説明を更新**
   - 変数を追加/削除したら説明テーブルも更新
   - 必須/オプションを明確に

3. **使用例を追加**
   - 実際の使用シナリオに基づく例
   - 基本的な使用と完全な使用の両方

4. **「テンプレート使用時の注意」セクションを削除**
   - 実際のテンプレートには不要
