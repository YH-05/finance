# Phase 0: 投資哲学の注入 — 設計メモ

> 作成日: 2026-02-10
> 前提: [CAGR推定フレームワーク](cagr_estimation_framework.md) のPhase 0に対応

## 1. 背景と制約条件

### 1.1 Phase 0 の位置づけ

CAGR推定フレームワークにおいて、Phase 0（投資哲学の注入）は全Phaseの前提条件である。競争優位性からCAGRへの接続に投資判断（目利き、投資哲学）が入るため、ここが模倣の核心となる。

### 1.2 模倣対象アナリスト（KY/吉沢）の制約

| 項目 | 状況 |
|------|------|
| 対象人数 | 1名 |
| 過去の「良い」レポート | なし |
| AIレポートへのフィードバック | あり（5銘柄31件の評価+コメント） |
| 哲学の言語化 | 完全ではないが、フィードバックから逆抽出可能 |
| PoC対象銘柄 | 未定 |

### 1.3 アプローチの選定

| アプローチ | 方法 | 可否 | 理由 |
|-----------|------|------|------|
| 明示的ルール | KYがルールを記述 | △ | KY自身が完全に言語化できていない |
| 暗黙的few-shot | KYの過去レポートを複数読ませてパターンを模倣 | × | 過去レポートが存在しない |
| **フィードバック駆動抽出** | AIレポートへの批評からルールを逆抽出 | **◎** | 5銘柄31件の評価+コメントが蓄積済 |

**結論**: 第3のアプローチ「フィードバック駆動抽出」を採用。`judgment_patterns.md` が実現可能性を証明済（12ルールがフォーマライズされ、各ルールに良い例/悪い例付き）。

## 3. 却下・低評価パターンの体系化（10-30%）

`judgment_patterns.md` セクション3から7つの却下パターンを整理。AIへのルール注入時、**「こうするな」の具体例**として機能する。

| パターン | 確信度 | 本質 | 典型例 |
|---------|--------|------|--------|
| **A: 結果を原因と取り違え** | 却下〜30% | 結果（シェア、成長率）を優位性と提示 | MNST#1: 高い市場シェアを優位性とする |
| **B: 業界共通で差別化にならない** | 20-30% | メガファーマ共通、多くの企業が言っている | LLY#6: グローバル医療機関ネットワーク |
| **C: 因果関係の飛躍** | 20-30% | 因果チェーンが3段階以上 | MNST#5: 低所得→コンビニ購買率→粘着力 |
| **D: 定性的で定量的裏付けなし** | 10-30% | 数値・比率・対競合比較がない | COST#2: 「劇場的価値」による差別化 |
| **E: 事実誤認** | 10%(即却下) | 財務指標の誤認 | MNST#6: OPMをGPMと誤認 |
| **F: 戦略を優位性と混同** | 30-50% | 意思決定を構造的力と混同 | ORLY#2: ドミナント出店「戦略」 |
| **G: 純粋競合に対する優位性不明** | 50% | 異業種比較のみで同業態比較なし | COST#3: SAMS CLUB/BJとの比較なし |

## 4. 高評価パターンの体系化（70-90%）

AIに「こうせよ」の具体例として機能する。

| パターン | 確信度 | 本質 | 典型例 |
|---------|--------|------|--------|
| **I: 定量的裏付けのある差別化** | 70-90% | 具体数値での競合比較 | COST#1: 退職率9% vs 業界20%+ |
| **II: 直接的なCAGR接続メカニズム** | 90% | 1-2ステップの因果+開示データ | COST顧客粘着力: 会員ミックス→ARPU→既存店売上 |
| **III: 能力>結果（プロセスの評価）** | 50-70% | 結果でなく能力を抽出 | CHD#1: 買収基準の明確化、人員投入、精査過程 |
| **IV: 構造的な市場ポジション** | 90% | 市場構造×企業ポジション合致 | ORLY#2: フラグメント市場×規模・密度・ネットワーク |
| **V: 競合との具体的比較** | 70% | 競合名を特定した差別化論証 | ORLY#1: AZO対比のDIFM補充頻度 |

## 5. スコア分布の定量的傾向

### 5.1 優位性スコア分布（N=31、5銘柄）

| 評価 | 確信度 | 件数 | 割合 | 特徴 |
|------|--------|------|------|------|
| かなり納得 | 90% | 2 | 6% | いずれもORLY（ルール11適用） |
| おおむね納得 | 70% | 8 | 26% | ルール4,7適用が多い |
| まあ納得 | 50% | 11 | 35% | **最頻値**: 方向性OK、裏付け不足 |
| あまり納得しない | 20-30% | 8 | 26% | パターンA,B,C違反 |
| 却下 | 10% | 2 | 6% | パターンD,E違反 |

### 5.2 CAGR接続スコアとの関係

- CAGR接続は優位性より全体的に高スコア（却下0件、70%以上が49%）
- KYの方針「優位性が100%あるものとして、CAGR接続への納得度評価」と整合
- 優位性とCAGR接続は**独立した2軸**で評価されている

### 5.3 銘柄別平均

| 銘柄 | 平均優位性 | 最高/最低 | 特徴 |
|------|-----------|----------|------|
| **ORLY** | **63%** | 90%/30% | 市場構造との合致が最高評価の要因 |
| CHD | 50% | 70%/30% | 能力vs結果の区別が明確 |
| LLY | 47% | 70%/30% | 業界共通能力を厳しく批判 |
| MNST | 40% | 70%/10% | シェア=結果の原則を厳格適用 |
| COST | 39% | 70%/10% | 最も分散が大きい。一般論を厳しく却下 |

## 6. feedback.md から得られるプロセス知見

`judgment_patterns.md` がルールの「What」を提供するのに対し、`feedback.md` はルール適用の「How」と「Why」を提供する。

### 6.1 フォーマット例が推論に与える影響

**発見（2025/11/13）**: MSFT/TSMCのフォーマット例をfew-shotとして与えた結果、LLYのREASONING段階で批判的推論が出力された。

KYの反応:
> 「LLYの例を与える（＝フォーマット規範として参考にせよ）と、MSFT/TSMCに批判的な推論を行うと考えてよいですか？」

チームの回答:
> 「Yesです。...良質なサンプルを与えてやることで納得感は高まると思います。」

**Phase 0への示唆**: few-shotとして与えるフォーマット例の選定が、AIの推論品質を大きく左右する。ただし、REASONING段階の出力は確率的であり、銘柄間で大きく異なる不安定性がある（2025/11/14の指摘）。

### 6.2 評価方法論の確定（2025/11/4）

KYとチーム間で以下が合意:

1. **コメント > グラデーション評価**: テキストコメントが優先。グラデーション（10%〜90%）は参考値
2. **優位性とCAGR接続は独立評価**: 「優位性が100%あるものとして、CAGR接続への納得度評価」
3. **グラデーション評価は継続**: 事後的にプロンプト側で調整も可能なため

### 6.3 改善サイクルの思想

KYの核心的期待（2025/11/13）:
> 「常に、上記を通じてphase1のクオリティを上昇 => phase2という改善サイクルを経るべき」

Phase 0を一回で完成させるのではなく、**Phase 1→Phase 2のフィードバックループでPhase 0のルールを継続的に精緻化する**ことを意味する。

### 6.4 KYが求める「警鐘機能」（2025/11/17）

> 「②の積み重ねから拡大解釈するのでなく、①の妥当性を再検討させるような仕組み考えられないのでしょうか？」
> 「FMやANが既存判断に縛られて、当初の合理性を欠いていることへの警鐘的機能ほしいです」

ルール12の実装要件。AIに**確認バイアスの検出器**としての役割を求めている。

### 6.5 REASONING段階の不安定性（2025/11/14）

KYの指摘:
- REASONING段階の指示内容・構成が銘柄ごとで大きく異なる
- LLYでは「考え直す」との見解のもと批判が続出（MSFT/TSMC例の影響）
- 一方、STEP3では推論での批判が消えて好意的に拡大解釈（推論→最終出力の断絶）

**Phase 0への示唆**: ルール注入はREASONING段階だけでなく、最終出力段階にも効かせる必要がある。

## 7. ブランド力評価フレームワーク

KYはMNSTフィードバックでブランド力に関する独自の4類型を展開。Phase 0で「ブランド力」を優位性として扱う際の判断基準となる。

| 類型 | 評価対象 | 具体例 |
|------|----------|--------|
| **1. ポートフォリオ運営力** | ブランドのマネジメント力 | ロレアル、ロリンズ、アトラスコプコ |
| **2. 知名度・認知** | デファクト地位 | MSFT, AAPL, MSCI, SBUX |
| **3. 価値維持・通貨代替** | 耐久性・交換価値 | AAPL, エルメス, ロレックス, マルボロ |
| **4. キーパーソンのC-suite化** | 経営コミットメント | コーチ/タペストリー, AAPL |

KY自身の自己評価:
> 「外見的な評価が多く、背後要因をもとにした評価が少ない印象」

ルール1（能力>結果）をブランド力にも適用しようとする姿勢を示している。

## 8. Phase 0 の設計提案

### 8.1 二層構造

```
Phase 0: 投資哲学の注入
│
├── 層1: 明示的ルール（12ルール + 却下/高評価パターン）
│   ├── ゲートキーパー: ルール9（事実誤認即却下）、ルール3（相対的優位性）
│   ├── 定義: ルール1,2,6,8（優位性とは何か/何でないか）
│   ├── 品質基準: ルール4,7,10,11（裏付けの質）
│   └── CAGR接続: ルール5,12（独立評価、①主②従）
│
└── 層2: 暗黙的参照（few-shot）
    ├── フォーマット例（MSFT/TSMC形式）
    ├── 高評価例（ORLY#2, ORLY#5）
    └── KY評価付きペア（良い例/悪い例）
```

### 8.2 適用タイミング

```
Phase 0 ルール
    │
    ├── REASONING段階 → 層1: 明示的ルールをシステムプロンプトとして注入
    │                   層2: few-shotをコンテキストとして提供
    │
    ├── 最終出力段階 → 層1: ゲートキーパールール（9,3）による自己検証
    │                  12ルールに対する適合チェック
    │
    └── 警鐘機能    → ルール12: ①期初レポートの前提への批判的再検証
                      確認バイアス検出器としての役割
```

### 8.3 改善サイクル

```
Phase 0 (v1: 12ルール)
    ↓
Phase 1: AI レポート生成（ルール適用）
    ↓
Phase 2: KY 評価・フィードバック
    ↓
ルール精緻化（追加/修正/優先度変更）
    ↓
Phase 0 (v2: 改善版ルール)
    ↓
  ...繰り返し
```

## 9. 未解決の論点

| # | 論点 | 説明 | 優先度 |
|---|------|------|--------|
| 1 | **12ルールの過不足** | 判断を再現するのにこの12ルールで十分か？ 追加/統合すべきものはないか？ | 高 |
| 2 | **ルールの優先順位** | PoCで全12ルールを同時適用するか、段階的に投入するか？ | 高 |
| 3 | **PoC対象銘柄の選定** | 既存5銘柄の再試行か、新規銘柄か？ スコア分布から ORLY（高評価）or COST（低評価）が候補 | 高 |
| 4 | **品質差の測定方法** | ルール適用前後の品質差をどう定量化するか？（KYの再評価が前提） | 中 |
| 5 | **REASONING段階の安定化** | 確率的不安定性への対処。ルール注入が最終出力まで貫通する仕組み | 中 |
| 6 | **ブランド力評価の体系化** | KYの4類型をルール化して注入すべきか？ KY自身が「背後要因の評価不足」を認識 | 低 |
| 7 | **確度のCAGRパラメータ統合** | LLYとCHDで確度の扱いが異なった問題（feedback.md 2025/11/17）への対処 | 中 |
