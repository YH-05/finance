## 議論1: 12ルールの過不足（設計メモ 論点1）

### 検討事項

| # | 論点 | 結論 |
|---|------|------|
| 1 | ルール1（能力≠結果）とルール8（戦略≠優位性）の重複可能性 | **別の視点として分離維持** |
| 2 | 「優位性の持続期間・耐久性」ルールの欠落 | **追加しない。** KYが言及していない以上、含めるべきでない |
| 3 | ブランド力4類型（設計メモ セクション7）のルール化 | **ルール化はしない。** ただし注入はする（議論3参照） |

### 確立された原則

> **ルール体系の範囲 = KYの暗黙知の表出のみ。**
> KYは自らの投資判断を体系化しているわけではなく、情報を取り込み、解釈し（暗黙知）、投資判断をアウトプットしている。
> そのため、フィードバックに現れた判断パターンはすべて暗黙知の表出であり、注入対象となる。
> 一方、分析者として「あるべき」ルールを追加することはしない。

### 結論

- 現行12ルールで確定
- 過不足の判断基準:「良い投資分析に足りているか」ではなく「KYの31件の評価を再現できるか」
- 今後KYから新たなフィードバックが出た場合のみ追加（改善サイクルで対応）

## 議論2: ルールの優先順位（設計メモ 論点2）

### 検討事項

| 方式 | メリット | デメリット |
|------|---------|-----------|
| 全12ルール同時投入 | KYの判断を忠実に模倣 | どのルールが効いたか分離できない |
| 段階的投入 | 各ルールの寄与度を測定可能 | 模倣精度低下、KYの評価負荷増大 |

### 結論

- **全12ルール同時投入**
- 理由: どのルールが効くかは対象企業のビジネスに大きく依存し、大量の企業レポートに対してフィードバックが得られず横比較が困難なため、分離実験は非現実的
- KYから新たなフィードバックが得られた場合はルール追加する

## 議論3: ブランド力4類型の扱い

### 背景

KYはMNSTフィードバックでブランド力に関する独自の4類型を展開（設計メモ セクション7）:

| 類型 | 評価対象 | 具体例 |
|------|----------|--------|
| 1. ポートフォリオ運営力 | ブランドのマネジメント力 | ロレアル、ロリンズ、アトラスコプコ |
| 2. 知名度・認知 | デファクト地位 | MSFT, AAPL, MSCI, SBUX |
| 3. 価値維持・通貨代替 | 耐久性・交換価値 | AAPL, エルメス, ロレックス, マルボロ |
| 4. キーパーソンのC-suite化 | 経営コミットメント | コーチ/タペストリー, AAPL |

KY自身の補足: 「外見的な評価が多く、背後要因をもとにした評価が少ない印象」

### 結論

- 独立したルールとしては追加しない
- **ルール1（能力≠結果）の補足として常時注入する**
- KY本人が「未完成」と評していても、それ自体が暗黙知の表出であり注入対象（議論1の原則を適用）
- 条件分岐なしでシステムプロンプトに含め、ブランド力が論点になった場合にAIが参照できるようにする

注入形式:
```
ルール1: 能力・仕組み ≠ 結果・実績
  └─ ブランド力の場合、以下の視点で評価する：
     1. ポートフォリオ運営力（ブランドのマネジメント力）
     2. 知名度・認知（デファクト地位）
     3. 価値維持・通貨代替（耐久性・交換価値）
     4. キーパーソンのC-suite化（経営コミットメント）
     ※ 「外見的な評価でなく、背後要因をもとに評価する」
```

## 議論4: アーキテクチャ（RAG/Difyナレッジ）

### 背景

AIはKYの競争優位性/CAGR接続の判断をRAGやDifyのナレッジ機能で参照する設計を想定。

### 懸念

RAGの検索は確率的であるため、常に適用すべきルールが取得されないリスクがある。

| カテゴリ | 取得漏れの許容度 |
|---------|----------------|
| A. ゲートキーパー（ルール9, 3） | **許容不可** |
| B. 優位性の定義（ルール1,2,8,6） | 低 |
| C. 裏付けの質（ルール4,7,10,11） | 中（企業依存で全部は該当しない） |
| D. CAGR接続（ルール5,12） | 低 |

### 結論: ハイブリッド構成

```
システムプロンプト（常時注入）
├── ゲートキーパー: ルール9, 3
├── 評価の枠組み: ルール5, 12（CAGR接続は常に独立評価）
└── 「ナレッジを必ず参照せよ」の指示

RAG/ナレッジベース（文脈に応じて検索）
├── 優位性の定義ルール（1,2,6,8）+ 具体例
├── 裏付けの質ルール（4,7,10,11）+ 具体例
├── ブランド力4類型
├── 却下パターン7種 + 高評価パターン5種
└── KY評価付きfew-shot（ORLY, COST, MNST等）
```

- Difyの詳細設計は後日詰める
- ナレッジのチャンク分割はルール単位で切り、検索精度を確保

## 議論5: PoC対象銘柄の選定（設計メモ 論点3）

### 方針

- **新規銘柄（アウトオブサンプル）** を使用
- 既存の他アナリスト執筆レポートからPoC対象を選定する
- セクターに依存しない投資判断を検証するため、既存5銘柄（CHD, LLY, MNST, ORLY, COST）とは異なるセクターが望ましい

### 選定基準

| 基準 | 内容 |
|------|------|
| アウトオブサンプル | KYが評価していない銘柄 |
| セクター非依存 | 既存5銘柄のセクター（消費財・ヘルスケア・自動車部品小売）と異なる |
| KYの知見 | 不要。銘柄への事前知見がなくてもフィードバック可能 |
| レポート | 他アナリストが執筆済みのレポートを使用する |

### CAGR推定フレームワークとの対応

CAGR推定フレームワーク（セクション3「登場人物と入力の構造」）の構造に対応:
- **入力アナリストレポート**: 他アナリストの執筆済みレポート → Phase 1で主張を抽出
- **10-K / 10-Q**: 可能な範囲で事実層として使用（非米国株の場合は代替ソース）
- **模倣対象アナリスト（KY）**: Phase 0で注入済みの12ルール+補足で判断

## 議論6: 品質差の測定方法（設計メモ 論点4）

### 検討した方式

| 方式 | 内容 | 採否 |
|------|------|------|
| A: 同一銘柄で2版比較 | ルール未注入版と注入版をKYが評価 | 不採用 |
| B: 既存5銘柄平均との比較 | ベースラインとの定量比較 | 不採用 |
| **C: テキストフィードバックによる定性評価** | 却下パターン出現頻度で測定 | **採用** |

### 目的の明確化

> **AIがKYの判断パターンに沿って議論すること**がプロジェクトの目的。
> チェックリスト的な排除（却下パターンをゼロにする）ではなく、KYの思考プロセスの再現を目指す。

### 測定フレームワーク

| 観点 | 測定方法 |
|------|---------|
| 却下パターン（A〜G）の出現 | KYのコメントを7パターンに分類し出現数をカウント |
| 高評価パターン（I〜V）の出現 | 同様にカウント。ルール注入により増加すれば改善の兆候 |
| 新規パターンの出現 | 既存7+5パターンに該当しないコメント → ルール追加の候補 |

### 成功/失敗の指標

| 指標 | 内容 |
|------|------|
| 成功 | KYが「自分の考え方に沿っている」と認識できる |
| 失敗 | KYのコメントに却下パターン（A〜G）が頻出する、または根本的なアプローチの乖離が指摘される |
| 副次的成果 | 新規パターンの発見 → 改善サイクルでルール追加 |

### CAGR推定フレームワークとの対応

CAGR推定フレームワーク（セクション5「今後の検討事項」）に記載の「模倣対象アナリストによるAIレポートの評価プロセスの設計」に該当。方式Cにより「納得するかどうか」の評価を却下/高評価パターンで構造化した。

## 議論7: REASONING段階の安定化（設計メモ 論点5）

### 問題の本質

設計メモ セクション6.5で指摘された2つの課題:

| 課題 | 内容 |
|------|------|
| 推論→出力の断絶 | REASONING段階で批判的に推論しても、最終出力で批判が消え好意的に拡大解釈される |
| 銘柄間の不安定性 | few-shot例の影響度が銘柄ごとに大きく異なる |

### 決定したワークフロー

以下の4ステップで推論の断絶を構造的に解決する:

#### ステップ1: 主張とファクトの分離・紐付け

- アナリストレポートから**競争優位性の主張**を抽出
- 10-K/10-Q等の情報から**ファクト**を抽出
- 主張と根拠を明確に紐付ける
- 可能であればチェック機構を入れる（引用数値の実在確認）

CAGR推定フレームワーク Phase 1（素材の整理）に対応:
- 10-K/10-Q → 事実層の抽出
- アナリストレポート → 見解層（主張）の抽出

#### ステップ2: 構造化出力の徹底

各競争優位性・CAGR接続について、推論の根拠を明確にした構造化出力を必須とする:

```
競争優位性候補 #N: [主張の内容]

ルール適用結果: [該当ルールの具体的内容を記述]
理由: [根拠情報（例: 2023年10-K Item 7 MD&Aの記載）を含めた判断理由]
確信度: [X%]
```

- KYの観点に沿わない主張も**削ぎ落とさない**
- ルール適用結果・理由・確信度をセットで必ず出力し、KYが「AIの判断プロセス」と「自分の判断」を比較できるようにする

#### ステップ3: ファクトチェック

情報ソースの階層を設け、検証可能な範囲で検証する:

| 優先度 | ソース | 対象 | 信頼性 |
|--------|--------|------|--------|
| 1 | SEC filings (10-K, 10-Q) | 米国株 | 最高 |
| 2 | 各国の法定開示（有報、Annual Report等） | 非米国株 | 高 |
| 3 | IR資料、決算説明会資料、投資家向けプレゼン | 全般 | 中 |
| 4 | Bloomberg/Reuters等の公開財務データ | 全般 | 中 |
| 5 | アナリストレポート内の数値（自己参照） | 全般 | 低 |

**検証不可の場合は必ず明示する（マスト）:**

```
主張: OPM 35%で業界トップ
検証: ○ 一致（ソース: 2025年度 Annual Report, p.47）
信頼度: 高

主張: 市場シェア25%
検証: △ 未検証（法定開示に該当データなし、アナリストレポート内の記述のみ）
信頼度: 低
```

#### ステップ4: 検証エージェントによる客観的検証

AIが構造化出力→レポートを作成したあと、**別のエージェント**が客観的に主張・観点を検証する。

| 観点 | 内容 | 優先度 |
|------|------|--------|
| KYの競争優位性ルール適合 | 12ルール + 却下パターンA〜Gとの照合 | マスト |
| CAGR接続の検証 | ルール5（直接的メカニズム）、ルール12（①主②従） | マスト |
| ファクトチェック | 引用数値の正確性、検証不可の明示 | マスト |
| 将来拡張枠 | 他アナリストの観点、業界固有の視点等 | 後日追加 |

検証観点は今後拡大可能だが、マストで入れるのはアナリストの**競争優位性・CAGR接続の観点**と**ファクトチェック**。

## 議論8: フィードバックを引き出しやすい仕組み

### 背景

改善サイクル（設計メモ セクション8.3）を回すには、KYから効率的にフィードバックを得る仕組みが必要。

### 課題

| 課題 | 影響 |
|------|------|
| KYの時間的負荷が大きい | レポート全体を読み込む必要がある |
| フィードバックの粒度がばらつく | 気になった箇所だけコメントし、問題ない箇所はスキップされる |
| 暗黙知の引き出しが受動的 | KYが自発的に言語化した部分しか取れない |

### 設計原則

- **KYの負荷を最小化する**
- **不同意の瞬間に暗黙知が出る**性質を活用する
- 提供形式はMarkdownまたはWord（静的ドキュメント。動的な回答収集は不可）

### 決定したフィードバックテンプレート

#### 主張ごとのフィードバック（3段階）

```markdown
### 競争優位性候補 #3: ベンダー交渉力による原価優位

根拠: 調達規模に起因し競合が容易に再現できない構造的優位
情報源: 2023年10-K Item 7 MD&Aにおける調達コスト開示、
        同業A社・B社のAnnual Reportとの比較

---

**納得度:**  10% / 30% / 50% / 70% / 90%  ← 丸をつける

**該当する質問に一言お願いします（1文で十分です）:**

- 納得しない場合 → 一番引っかかる点は？
- どちらとも言えない場合 → 何があれば納得度が上がる？
- 納得する場合 → 他の企業でも同じことが言えない理由は？

回答:


**補足（任意）:**

```

#### レポート末尾の全体評価

```markdown
## 全体フィードバック

**レポート全体として、あなたの考え方に沿った議論ができていますか？**

□ 概ね沿っている  □ 部分的に沿っている  □ 沿っていない

**最も良かった主張の番号:**  #___
**最も問題のある主張の番号:**  #___

**その他コメント（任意）:**

```

### 蓄積されるデータ

| 蓄積されるデータ | 源泉 |
|----------------|------|
| 確信度分布 | 納得度の選択 |
| 却下理由・判断根拠 | 「一番引っかかる点は？」への回答 |
| 暗黙知（求める証拠の種類、比較基準） | 「何があれば納得度が上がる？」への回答 |
| 相対的優位性の基準 | 「他の企業でも同じことが言えない理由は？」への回答 |
| 全体的な方向性の適合度 | 全体評価欄 |
| KYが重視する主張の優先度 | 最良/最問題の番号 |
| 想定外の洞察 | 任意の補足コメント |

## 議論9: 確度のCAGRパラメータ統合（設計メモ 論点7）

### 問題

LLYとCHDで「確度」のCAGRパラメータへの取り込み方が異なった（feedback.md 2025/11/17）:

| 銘柄 | 確度の扱い |
|------|-----------|
| LLY | 達成確度とパラメータ寄与を**分離**（確度は確度、寄与幅は寄与幅として別出力） |
| CHD | 確度をパラメータ寄与に**統合**（確度が低ければ寄与幅も縮小） |

両者とも同じCDNSフォーマットを参考にしているのに結果が異なった。

### KYの見解

> 競争優位性はパラメターへの寄与のみならず、その確度にも反映させるべき。
> 特に競合に対するシェア上昇をCAGRのパラメターとした場合は、シェア上昇幅および達成確度とも競争優位性に裏打ちされているべき。

KYは**CHD方式（確度を寄与に統合）を支持**。

### 結論

**ルール5の補足として扱う。新規ルール追加は不要。**

理由:
1. **抽象度が異なる**: 12ルールは「何を評価するか」の判断基準。確度統合は「評価結果をどう表現するか」の出力形式であり、ルール5の運用指針に収まる
2. **ブランド4類型との一貫性**: 既存ルールの補足として吸収できるものはルール数を増やさないパターンを踏襲
3. **ルール体系の安定性**: 既存12ルールのいずれにも該当しない新たな判断軸が出たときのみ新規ルール追加

注入形式:
```
ルール5: 直接的メカニズム + 検証可能性
  └─ CAGRパラメータの表現:
     寄与幅と達成確度は統合して反映する。
     競争優位性の強度が高ければ寄与幅・確度とも高くなり、
     弱ければ両方下がる。分離して別出力しない。
     例: シェア上昇をCAGRパラメータとする場合、
         上昇幅と達成確度の両方が競争優位性に裏打ちされていること。
```

### ルール追加の基準（新規確立）

> 既存12ルールの補足で吸収できるものは補足に留める。
> いずれの既存ルールにも該当しない新たな判断軸が出たときのみ、新規ルールとして追加する。
